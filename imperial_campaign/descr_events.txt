; Be careful with changing something in this file:
; - events are used in EDB and script, some also in EDCT, EDA, DFS, descr_mercenaries.
; - there may be path-dependence: an event from here may be used in the script to unlock / trigger other events.
; - change of the name of the event must be accompanied by the change in historic_events.txt (otherwised you'd get a crasha when the event happens).
; - events may have their special, dedicated pictures - you need to add them in 5 diffrent directories (the same file is enough, no need to change name).
; - some events in this file (they begin with ;;;;;; ) are only for a modder information purpose as they are defined in script.txt (they usually depict reforms).
; - events "INFO_" should not be here - all should be in the script on even turns. They should never be used in other files: they are just to inform the player about mechanics.
;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;
;
; WHEN AN EVENT HAPPENS / SHOWS UP:
;   date 0 is 1132 (and date 1 is 1133)
;   events in this file pop-out on odd turns (summer): turn = ( date x 2 ) + 1
;      (except of 0 - doesn't show at all - it may be used for triggering earlier events)
;   you can create events happening on even turns (winter) using the script.txt: turn = I_TurnNumber + 1
;   events scheduled for the same turn are shown-in game (to the player - that icon falling on the left side of the screen at the beginning of a turn):
;   - first: in the reverse order if coded in this file (second before the first etc.)
;   - second: in the normal order if coded in campaign.txt
;
;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;
;
; MERCENARIES - you can use "events" only from this file - keep it in mind when you transform something into the script.txt
;
; so: DON'T TRANSFROM ARMOUR-RELATED EVENTS to script. Leave them here.
;
;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;
;
; CODING
; Events are followed by a category, and a label
; labels are used as lookups into historic_events.txt for title and body strings,
; and also as image names for the event picture
;
; Currently supported categories are:
; - counter  - increases a counter
; - historic - increases a counter AND generates a pop-out window (message) for the player
; - volcano  - triggers a volcano if there is one on the map at the position specified
; - plague   - triggers a plague in settlements at the specified positions
; - emergent_faction - triggers the emergence of the given faction. This faction must be marked as emergent in descr_strat.txt.
; - fire - 
;
; event descriptions are followed by a date (year as offset from starting date) or a date range
; and either a list of positions, or list of regions.  Each position/region specification must appear on
; a seperate line below the date
;
;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;
